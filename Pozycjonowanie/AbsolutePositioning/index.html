Absolute positioning

Elements that are relatively positioned remain in the normal flow of the document.

Element that is absolutely positioned is taken out of the flow - other elements are positioned as if it did not exist.

This absolutely positioned element is positioned relative to its nearest positioned ancestor (the nearest is that is not static). If a positioned ancestor doesn't exist, the initial container is the html element. This is more accurately called the element's positioning context.

When positioned and non positioned elemnts overlap, positioned elements display over non positioned.

When we have two positioned elements, the second element in the source order is displayed over the first one.

We can change the display order using the z-index property.

The z-axis is an imaginary line that runs from the surface of your screen, towards your face. Elements with positive z-index sits higher up the stack; elements with negative z-index values are lower down the stack.

By default, positioned elements all have a z-index of auto, which is 0.
